Pellet's MCP server is bundled with [`@pelletnetwork/cli`](/docs/wallet-cli). Run `pellet mcp` to start a stdio MCP server that exposes `pellet_status` and `pellet_pay` to any agent runtime.

→ **[Full integration guide is at Wallet · MCP](/docs/wallet-mcp).**

## TL;DR

```json
{
  "mcpServers": {
    "pellet": {
      "command": "npx",
      "args": ["-y", "@pelletnetwork/cli", "mcp"]
    }
  }
}
```

Works with Claude Code, Cursor, Cloudflare Agents, and the Anthropic API directly via stdio MCP.

## Why isn't there a separate MCP for OLI?

OLI is read-only and CORS-open at `/api/oli/*`. Any agent that can `fetch` can hit OLI directly — no MCP wrapper required. The skill manifest at [pellet.network/skill.md](/skill.md) teaches the agent the endpoints + when to use them.

If you want an MCP-flavored OLI surface specifically (e.g. for runtimes that only invoke MCP tools, never raw HTTP), open an issue at [github.com/pelletnetwork/pellet/issues](https://github.com/pelletnetwork/pellet/issues) and we'll bundle one with the CLI.
